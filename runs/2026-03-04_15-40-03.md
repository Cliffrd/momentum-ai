## Analysis

**KEY_THEMES:**

- **Export functionality is a critical bottleneck** — Enterprise users like Sarah spend 2+ hours weekly on manual exports; 23 support tickets cite row limits; 41 feature requests for CSV improvements. This is a workflow pain point, not a nice-to-have.
- **Onboarding fails to surface core value** — James nearly churned because bulk actions were invisible for 3 weeks; 31 tickets ask basic "how do I add a user" questions; 22 requests for an onboarding checklist. Users aren't discovering what they're paying for.
- **Permissions/user management creates friction at scale** — Sarah's team has wrong access levels and can't self-serve; 38 requests for bulk user management. This blocks enterprise expansion and creates IT dependency.
- **HubSpot integration is the loudest unmet need** — 44 support inquiries + 67 feature requests. This is table-stakes for the sales/marketing workflow your users likely have.
- **Performance issues erode trust** — 18 Firefox-specific complaints suggest uneven QA; slow dashboards undermine the "quick insights" promise.

**OPPORTUNITIES:**

- **Scheduled/templated exports with custom formatting** — Eliminates Sarah's 2-hour Monday ritual, directly addresses enterprise retention — *Medium effort*
- **Guided onboarding with progressive feature disclosure** — Reduces "accidental discovery" churn risk like James, cuts basic support tickets — *Medium effort*
- **HubSpot integration** — Highest volume request, likely blocking deals and expansion — *High effort but high signal*
- **Bulk user management + permission presets** — Unblocks enterprise self-service, reduces IT dependency complaints — *Medium effort*
- **Raise export row limit (quick win)** — 23 tickets/month is a bleeding cut; likely a config change — *Low effort*

**HYPOTHESIS:**

"If we ship scheduled exports with boss-friendly templates, then enterprise power users (like Sarah) will reduce manual reporting time by 80%, because the current export flow forces repetitive manual formatting that creates weekly friction and risks champion burnout."

**PRD_OUTLINE:**

**Problem Statement:** Enterprise users spend hours weekly exporting data and reformatting it manually. Current export is limited (1K rows), unscheduled, and format-inflexible—creating friction that threatens retention and expansion.

**Target User:** Enterprise "champion" users (100+ seats) who report to executives and need recurring, formatted data exports.

**Success Metrics:**
- Reduce average export-related support tickets by 50% within 60 days
- 30% of enterprise accounts adopt scheduled exports within 90 days
- NPS increase among enterprise segment (+5 points)

**Proposed Solution:**
- Scheduled exports (daily/weekly/monthly) delivered via email or cloud storage
- Customizable templates (column order, naming, date formats) saved per user
- Raise row limit to 50K (or async generation for larger sets)

**Open Questions:**
- What file formats beyond CSV are needed? (Excel with formatting? Google Sheets direct?)
- Should templates be shareable across team, or user-specific?
- Do we need an approval workflow for scheduled exports containing sensitive data?

## TL;DR

**Raise the export row limit immediately** — it's a config-level fix that stops the bleeding (23 tickets/month) while you build the larger scheduled export solution.
